Module Caqti_metadata

module Caqti_metadata: sig .. end
Information about a database, its backend, and its query language. This is static data supplied by the backend to aid the application in dealing with differences between database systems.
type dialect_tag = private [> `Mysql | `Pgsql | `Sqlite ]
A tag used for easy dispatching between query languages.
type sql_dialect_tag = [ `Mysql | `Pgsql | `Sqlite ]
Subtype of the above which includes only SQL dialects.
type parameter_style = private [> `Indexed of int -> string | `Linear of string | `None ]
How parameters are named. This is useful for SQL since the difference between dialects typically have an intrusive effect on query strings. This may also be useful for non-SQL languages which support some form of variables or placeholders.
  • `None means that non the following parameter styles apply, or that the backend does not support parameters at all.
  • `Linear s means that occurrences of s bind to successive parameters.
  • `Indexed f means that an occurrence of f i represents parameter number i, counting from 0.

Information about database, backend, and query language. For the meaning of the fields, see the corresponding parameters to Caqti_metadata.create_backend_info.
val create_backend_info : uri_scheme:string ->
       ?dialect_tag:dialect_tag ->
       ?parameter_style:parameter_style ->
       ?default_max_pool_size:int ->
       describe_has_typed_parameters:bool ->
       describe_has_typed_fields:bool -> unit -> backend_info
For use by backends to create a descriptor for their query languages.
uri_scheme : The URI scheme this backend binds to.
dialect_tag : The Caqti_metadata.dialect_tag describing the SQL dialect or other query language. Omit it if non of the cases apply.
parameter_style : How to represent parameters in query strings.
default_max_pool_size : The suggested pool size for this backend. Unless the backend is special like preferring a single connection, leave this out.
describe_has_typed_parameters : True iff the describe function is capable of supplying information about parameter types.
